Estructura de archivos

📁 supabase-agent/
  ├── agent.ts           # exporta runAgent(prompt, history) → AsyncGenerator<string>
  ├── server.ts          # Express: rutas HTTP + SSE + leer/guardar historial
  ├── public/
  │   ├── index.html     # estructura del chat
  │   ├── style.css      # diseño fiel a ui.md
  │   └── app.js        # lógica: enviar mensaje, consumir SSE, renderizar
  ├── chat-history.json  # generado en runtime (gitignored)
  └── .env

Rutas del servidor (server.ts)

GET /
Sirve public/index.html
GET /api/history
Devuelve el historial guardado en JSON
POST /api/chat
Recibe {"{message}"}, abre SSE, streama chunks del agente, guarda en historial al terminar
DELETE /api/history
Limpia el historial (botón "Nueva conversación")

Flujo de un mensaje

Browser POST /api/chat { message }  →  server.ts abre SSE, llama runAgent()  →  agent.ts query() con historial completo
                                                          ↓ chunks de texto via SSE
Browser renderiza cada chunk en tiempo real
server.ts al terminar → guarda par user/assistant en chat-history.json

Formato del historial (chat-history.json)

[
  { "role": "user", "content": "¿Cuántos usuarios hay?" },
  { "role": "assistant", "content": "Hay 142 usuarios registrados." }
]
