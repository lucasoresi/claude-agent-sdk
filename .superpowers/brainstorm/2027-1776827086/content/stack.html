¿Cómo querés agregar la UI?

El proyecto actual es un CLI en Node.js/TypeScript puro. Hay 3 formas de agregarle una interfaz web.

A

Express + HTML/CSS/JS vanilla

Agregar un servidor Express al mismo index.ts. Sirve una página HTML con el chat. La comunicación usa Server-Sent Events (SSE) para el streaming del agente. Sin frameworks extra.


Pros

  • Mínimas dependencias nuevas
  • Todo en un solo proceso
  • Fácil de entender

Cons

  • Sin hot-reload en desarrollo
  • CSS y JS escritos a mano
B

Next.js (React + API Routes)

Nueva app Next.js en una subcarpeta web/. El frontend React consume un API Route que internamente llama al agente SDK. Streaming vía ReadableStream.


Pros

  • Idéntico al diseño de IACA (v0)
  • Hot-reload, TypeScript nativo
  • Componentes reutilizables

Cons

  • Muchas dependencias nuevas
  • Dos procesos en desarrollo
C

Vite + React (frontend separado)

Frontend React con Vite (rápido, liviano). Backend: Express en el mismo repo. Más ligero que Next.js pero con la potencia de React.


Pros

  • Más liviano que Next.js
  • Build ultra-rápido
  • Separación clara frontend/backend

Cons

  • Requiere configurar ambos puertos
  • Más setup inicial que Express puro
